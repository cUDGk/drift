# DRIFT — 構想

## 1. 問題意識

クラウド LLM (ChatGPT / Claude / Gemini) は強力だが、**会話内容が必ず運営元に渡る**。
ロールプレイ、機微な相談、創作の試行錯誤、個人的な愚痴 — 外に出したくない会話は確実に存在する。

オンデバイス推論 (llama.cpp 等) はこの問題を原理的に解決するが、
小型モデル (3B〜7B) ではコンテキスト窓が狭く、会話が長くなるとすぐに破綻する。
「キャラが前のやり取りを忘れる」「設定を食い違える」「ロールプレイが続かない」。

**DRIFT はこの二つの壁 — プライバシーと長期一貫性 — を同時に越えることを目指す。**

## 2. 設計原則

### プライバシー第一
- 推論: 端末内で完結 (llama.cpp / NDK / arm64-v8a)
- 保存: アプリ専用領域のみ
- ネットワーク: モデル取り込み時を除き使用しない
- テレメトリ・クラッシュレポート・分析の類を持たない

### 記憶はユーザーのもの
- 会話履歴・キャラ記憶・要約は**全てローカルで、可視で、編集可能**
- ブラックボックスな「忘却」や「推測による改変」を避ける
- キャラが壊れた時、ユーザーが手で直せる

### モデル非依存
- GGUF フォーマットで差し替え自由
- 将来、より軽量で高性能なモデルが登場した時に、アプリ本体を変えずに移行できる
- 用途別の複数モデル (軽量日常 / RP / 専門) を同時に持てる

### 軽量で読みやすい実装
- WebView + JS で UI、ネイティブ層は推論ブリッジに集中
- 行数を増やすより減らす選択を優先

## 3. 核となるアイデア

### 3.1 擬似無限コンテキスト — 階層化メモリ

小さいコンテキスト窓で長期会話を成り立たせるため、記憶を複数の層に分けて管理する。

- **永続キャラ記憶**
  キャラクターやユーザーについて発見された事実・関係性・重要イベントを構造化して保持。
  毎回のプロンプトに必ず含める。

- **直近ターン**
  最近の会話を高解像度 (生ログ) のまま保持。

- **中期要約**
  直近ターンから溢れた分を時系列でローリング圧縮。

- **長期アーカイブ**
  さらに古い要約をインデックス化し、必要になった時に検索で呼び戻す。

各層のトークン予算は固定配分で、合計がコンテキスト窓に収まるように動的に調整する。
詳細な容量配分・更新タイミング・要約戦略は実装フェーズで詰める。

### 3.2 キャラクター / ペルソナ

SillyTavern 的なキャラクターカード (名前 / 説明 / 性格 / シナリオ / 会話例) と、
ユーザー側のペルソナ (呼称 / 一人称 / 口調) を分離して管理する。

キャラには「好みのモデル」を紐付けでき、用途に応じて自動で切り替わる。

### 3.3 ユーザーによる記憶の掌握

以下を第一級機能として扱う:

- 記憶ビューア (階層化メモリの全層を閲覧)
- 記憶の手動編集
- 特定ターンのピン留め (自動圧縮・削除対象から除外)
- 全記憶のエクスポート / 削除

記憶機構をユーザーがコントロールできない限り、このアプリは「ただ性能の悪いチャット AI」になる。
ここは絶対に譲らない。

## 4. スコープ外 (現時点)

意図的に作らないもの:

- クラウド LLM 連携
- マルチデバイス同期
- 音声入出力 / 画像生成連携
- アカウント機能・ログイン
- 複数ユーザー

オンデバイスで軽量かつ高性能なモデルが広く使えるようになった時点で、
アーキテクチャの核 (ローカル推論 + 階層化メモリ + キャラ管理) を検証することに全力を注ぐ。

## 5. 類似プロジェクトとの立ち位置

- **SillyTavern** — 機能豊富だが基本はクラウドモデル依存。オンデバイス運用が弱い
- **ChatterUI / MLC Chat** — オンデバイス推論は動くが、長期記憶・キャラ管理が素朴
- **DRIFT** — オンデバイス推論 + 長期記憶 + キャラ管理 + モバイル最適化 の四点セット

## 6. 将来の分岐点

現状は 3B〜7B クラスの量子化モデルを前提に組むが、以下が到来した時に設計の一部は書き直す:

- **1B 台で現行 7B 相当の品質が出るモデル** — 階層化メモリの各層予算を拡張できる
- **MoE のオンデバイス実装が実用になる** — 専門モデル自動切替の粒度が上がる
- **端末側のベクトル検索が軽くなる** — 長期アーカイブの retrieval 精度を底上げできる

現時点の実装はこれらに追従しやすい形にすることを意識する。
